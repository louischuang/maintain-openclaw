maintain-openclaw

OpenClaw Service Helper

這份教育文件說明如何使用本專案安裝、維護、檢查，以及安全移除 OpenClaw 相關服務。它不是自動化入口，而是操作前的閱讀與確認清單。

專案用途

安裝

把 OpenClaw 的安裝步驟整理成文件與腳本，讓新環境可重複建置。

狀態檢查

檢查程序、服務、port、設定檔、版本與日誌，快速判斷是否健康。

維護

記錄更新、備份、回滾、排查與日常操作，降低維運成本。

移除服務

用 dry-run 先盤點 OpenClaw 相關資源，再經確認後停止、停用與刪除。

測試環境

目前可使用下列 VM 進行安裝、狀態檢查與移除服務流程測試：

cp .env.example .env
$EDITOR .env
source .env
ssh "$OPENCLAW_TEST_VM_SSH_TARGET"

操作提醒

登入後請先確認 `hostname`、`whoami`、`uname -a`。刪除服務前必須確認這台主機不是正式環境。

建議閱讀順序

  1. README.md：了解專案入口、測試 VM 與主要文件。
  2. AGENTS.md：了解代理人或維護者應遵守的工作規範。
  3. MVP.md：了解第一版要交付的安裝、檢查與移除能力。
  4. TODO.md：挑選下一步可執行工作。
  5. docs/remove-openclaw-services.md：執行移除服務前必讀。

刪除服務的安全流程

  1. 先用 dry-run 盤點，不做任何刪除。
  2. 檢查輸出中是否有 OpenClaw 相關 systemd、launchd、Docker、cron、程序或 port。
  3. 備份服務定義、設定檔與必要日誌。
  4. 確認要刪除的項目與目標主機。
  5. 只有在明確確認後，才使用 destructive mode。
./scripts/remove-openclaw-services.sh --dry-run

./scripts/remove-openclaw-services.sh \
  --confirm \
  --yes-i-understand-this-removes-openclaw-services

日常操作範例

檢查本機腳本語法

bash -n scripts/remove-openclaw-services.sh

在 VM 上測試 dry-run

source .env
scp scripts/remove-openclaw-services.sh "$OPENCLAW_TEST_VM_SSH_TARGET":/tmp/remove-openclaw-services.sh
ssh "$OPENCLAW_TEST_VM_SSH_TARGET" 'bash /tmp/remove-openclaw-services.sh --dry-run'

查看文件

open service-helper.html
